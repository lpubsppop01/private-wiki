# Swift 開発環境
```
macOS については公式動作環境があるだろうが、他の環境ではどうか？
まず Apple が提供している SDK は macOS 用と GNU/Linux 用。Windows 用はない。

Silver という非公式実装があり .NET, Java, Android, Cocoa で動作する。
Windows では Visual Studio 2015、macOS では Fire という IDE を使用して開発できる。
OSS ではなく無償のプロプラソフトなので将来的に有償化する可能性はある。

Swift for Windows という非公式実装も最近公開されたらしい。
https://osdn.jp/magazine/16/05/10/154000

Eclipse 用の Swift サポートである swiftclipse というのがあり、
一応これならマルチプラットフォームで開発できそうではある。

---------------------------------------------------------------

現状だと Silver が有用そう。
アプリ層とライブラリ層をしっかり分ければ、何かあっても最低限ライブラリは移植できるだろうし。

しかし公式環境も試さないといろいろ難しいか。
久々に Mac Book Pro を復活させるか…。

---------------------------------------------------------------

XCode と Silver を試した。
開発環境の安定性はどちらも今一歩。
Silver は VS な分キーバインドなどは有利だが、準拠している言語のバージョンがどうも古いっぽい？

---

Swift for Windows も試してみた。
ほぼ先端の 3.0 2016/04/18 ベースのため言語の機能的には文句なしなのだが、
おそらく Foundation が含まれてない？
build も使えなかった。
よほど慣れてる人でないとこれをベースに開発は厳しい印象。

これくらいなら Windows 10 の Windows Subsystem for Linux の上で
Linux 版を使用する方が健全な気がする。
```
