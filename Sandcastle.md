# Sandcastle
```
Sandcastle は .NET 向けの API ドキュメント生成ツール。
重い、使いにくいという印象だったのだが、今試してみたら割と悪くなかった。

インストールは chocolatey で OK。
環境変数が追加されるそうで、インストール直後に起動したら再起動して的なメッセージが表示された。

使い方はかなり簡単。
- プロジェクトファイルを新規作成。
デフォルトだとプロジェクトファイルと同じフォルダに Help というフォルダが作られて
ファイルが出力されるので注意。
- ビルドするファイル形式を選択。
HTML Help と Website を試してみた。どちらも問題ない。Website の方もサーバーなしで動作した。
- ツールバーからビルド。
Doxygen のときと比べるとやや時間がかかる気はする。

プロジェクトファイルを用意しておけばコマンドラインからビルドできるらしい。

.NET クラスライブラリヘルプ(HTML)をコマンドラインからサッと作れるようにする - lriki’s blog:
http://lriki.hatenablog.com/entry/2015/10/03/174801

---------------------------------------------------------------

テストコードから対応済みパターンのドキュメントを作る用途で考えていたのだが、
一応そういうこともできそう。
・[Visibility] で全オフにすれば継承メンバーは表示されなくなる。
・[Visibility]-[Edit API Filter] で対象を限定できる。
・コメントに <img src="..."/> タグで画像を埋め込める。data スキームも可。
これはわくわくする。
```
